Environment Preparation Lab

This course provides in-depth exposure to Red Hat OpenStack Platform director. The lab exercises guide you through key procedures and concepts. In this lab, you set up the environment and ensure that you have connectivity. Connection details and credentials have been assigned to you, so please let the instructor know if you do not have the necessary information.

Goals

  • Configure two basic virtual networks, provisioning and trunk networks.

  • Create a virtual machine to be used later as the undercloud node.

  • Create virtual machines to be used as the overcloud nodes.

Lab Assets

  • Access to a multi-CPU bare-metal system with at least 64 GB of memory

  • VMs on top of the bare-metal host to mimic a real-world cloud deployment

Requirements

  • Personal laptop for SSH client and web browser

1. Review Lab Environment

The following exercises prepare this host to run the undercloud and overcloud VMs:

  • Configure the necessary logical/virtual networks to represent real-world networks

  • Configure a VM that becomes your undercloud node

  • Configure multiple empty VMs that eventually become your overcloud nodes

  • Ensure connectivity of the undercloud node and verify the networks

If you are using these materials for self-paced learning, the instructions are designed for implementation upon a bare-metal host with a clean Red Hat Enterprise Linux 7.6 installation.

2. Verify Access to Host

As this machine is allocated to you for the duration of the course, you can use the SSH key provided by the instructor to access it.

  1. Copy the private SSH key to your laptop. For example, the private key may be presented to you as the student-id_ed25519 file. Copy it to your ~/.ssh directory and create the ~/.ssh/config file like that:

    Sample .ssh/config
    Host <your hostname>
User student
IdentityFile ~/.ssh/student-id_ed25519

  2. Check that you can access the host as the student user and perform sudo:

    client$ ssh <your given host>
[student@hypervisor ~]$ sudo -i
[root@hypervisor ~]#

5. Install Cockpit

Cockpit is a user-friendly web-based interface for administering servers. It allows monitoring system resources and adjusting configuration with ease.

  1. Install Cockpit to manage Virtual Machines and access console

    [root@hypervisor ~]# yum install -y cockpit cockpit-dashboard cockpit-machines

  2. Configure firewalld to allow access to port 9090

    [root@hypervisor ~]# firewall-cmd --add-port=9090/tcp
[root@hypervisor ~]# firewall-cmd --permanent --add-port=9090/tcp

  3. Start and enable the cock service

    [root@hypervisor ~]# systemctl enable --now cockpit

  4. Access to your hypervisor browsing to https://pool-XX-iad.lab.opentlc.com:9090

    Username is root and password r3dh4t1!

  5. Browse in the left side to Virtual Machines and review the VMs and Networks there.
